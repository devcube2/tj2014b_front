프로그래밍 언어의 정의
    - 정의
        cpu(중앙처리장치)가 이해할 수 있는 기계어(이진수) 코드를 생성해주는 언어
    - 필요한 이유        
        cpu(중앙처리장치)는 기계어(이진수)만을 해석할 수 있는데,
        기계어 프로그래밍은 인간의 사고로는 사용이 99.999% 불가능하다.
        그러므로 기계와 인간의 교점이 되는 새로운 언어가 필요하며
        그게 프로그래밍 언어이고, 필요한 이유이다.
    - 종류
        절차적 언어, 객체 지향 언어, 함수형 언어 3가지로 나뉜다.
        절차적 언어(C)
            - 코드의 실행 흐름이 순차대로 이루어진다.
            - 코드의 흐름이 직관적으로 다른 언어에 비해 흐름을 이해하기 쉬운편이다.
            - 코드의 한 부분에 문제가 생길 경우, 전체에 영향을 줄 가능성이 크다.            
        객체 지향 언어(JAVA)
            - 코드의 실행 흐름이 객체(기능)단위로 쪼개어진다.           
            - 절차적 언어에 비해 유지보수가 좀 더 용이하다. 
            - 객체의 설계가 중요하며, 규모가 커질수록 구조가 복잡해진다.
        함수형 언어(Clojure)
            - 전역 변수가 없고, 각 함수내에서 모든 변화가 끝난다.
              스레드 버그가 근본적으로 존재할 수 없는 언어이다.
            - 멀티스레딩 을 통한 대규모 시스템 구현에 적합하다.
            - 기존 언어에 비해 생소한 개념으로 익히기가 어렵다.

데이터 와 데이터 타입
    - 정의
        데이터: 이진수의 덩어리
        데이터 타입: 데이터의 종류를 결정
    - 필요한 이유
        파일(데이터)과 파일 확장자(데이터 타입)의 관계로 보면 된다.
        파일이 없으면 실체가 없는것이고, 파일 확장자가 없으면 정체를 알수없는 덩어리(데이터)에 불과하다.
        그러므로 서로 필수적인 관계이다.
    - 데이터 타입 종류
        C언어로 예를 들면 int, long, float, double, char 등이 있다.
        int, long : 정수 타입
        float, double : 실수 타입
        char : 문자 타입
